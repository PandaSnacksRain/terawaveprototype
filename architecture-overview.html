4. Integration Priority & Critical Path

Key Insight
The critical path is determined by data availability, not code complexity. If data exists and is accessible, integration is straightforward. If data must be captured fresh, scope expands significantly.

CRITICALMES Integration

Why: Work order data, station status, serial tracking - the backbone of everything

Data needed: WO number, serial, current op, station, cycle time

Blockers: API availability, real-time vs batch, data freshness

Effort: 3-6 weeks depending on API maturity

CRITICALAndon Capture

Why: If andons aren't captured digitally today, this is ground zero

Options: Physical buttons → PLC → API, or tablet-based pull

Blockers: Hardware install, PLC programming, network

Effort: 2-4 weeks if software only, 8-12 weeks if hardware

HIGHNC/Quality System

Why: NC creation, disposition tracking, MRB workflow

Options: Integrate existing system or build NC module

Blockers: Existing system API, or schema design if new

Effort: 2-4 weeks to integrate, 6-8 weeks to build

HIGHEquipment/Cal System

Why: PM schedules, calibration status, equipment history

Data needed: Equipment registry, cal dates, PM intervals

Blockers: Data quality, system API access

Effort: 2-4 weeks

MEDIUMHR/Training

Why: Certifications for responder assignment

Data needed: Person → cert mapping, expiration dates

Workaround: Manual cert table initially

Effort: 1-2 weeks

MEDIUMERP Integration

Why: Parts data, WO master, cost tracking

Deferrable: Can start with MES data only

Effort: 2-4 weeks when ready

8. Tech Stack Recommendation

Frontend

React + TypeScript - Component reuse across dashboards

TanStack Query - Data fetching, caching, real-time

Tailwind CSS - Matches prototype styling approach

Backend

Node.js/Express or Python/FastAPI - Fast development

PostgreSQL - Relational data, JSONB for flexibility

Redis - Real-time state, pub/sub for notifications

Infrastructure

Kubernetes - Container orchestration (likely existing)

API Gateway - Kong or AWS API Gateway

Message Queue - RabbitMQ or Kafka for async

Integration

Adapter Pattern - Abstract external system specifics

Event-Driven - Loose coupling, audit trail

Idempotent APIs - Handle retries gracefully

9. Risk Factors

Data Availability

If source systems don't expose the data we need, scope explodes. Validate API access early.

Change Management

New workflows = new habits. Supervisor buy-in and training are as important as code.

Real-Time Requirements

Andons need sub-second visibility. If MES only does batch updates, we need a workaround.

Scope Creep

Prototypes show the ideal. Production needs error handling, edge cases, offline mode.

Bottom Line

Best case (data exists, APIs available): 4-5 months with 3-4 engineers

Realistic case (some integration work): 6-7 months with 3-4 engineers

Worst case (building capture layer + NC system): 9-12 months

The prototypes cut 2-3 months off any timeline by eliminating requirements ambiguity.
